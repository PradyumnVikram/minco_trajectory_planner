Callback interface to provide an upper bound at the beginning of the current line search.

The \mbox{\hyperlink{namespacelbfgs_a25919c3755adc48965ba380d9254be39}{lbfgs\+\_\+optimize()}} function call this function to obtain the values of the upperbound of the stepsize to search in, provided with the beginning values of variables before the line search, and the current step vector (can be descent direction). A client program can implement this function for more efficient linesearch. Any step larger than this bound should not be considered. For example, it has a very large or even inf function value. Note that the function value at the provided bound should be FINITE! If it is not used, just set it nullptr.

Start a L-\/\+BFGS optimization. Assumptions\+: 1. f(x) is either C2 or C0 but piecewise C2;
\begin{DoxyEnumerate}
\item f(x) is lower bounded;
\item f(x) has bounded level sets;
\item g(x) is either the gradient or subgradient;
\item The gradient exists at the initial guess x0. A user must implement a function compatible with \mbox{\hyperlink{namespacelbfgs_a415529dd955b1806c101e02eabf575a4}{lbfgs\+\_\+evaluate\+\_\+t}} (evaluation callback) and pass the pointer to the callback function to \mbox{\hyperlink{namespacelbfgs_a25919c3755adc48965ba380d9254be39}{lbfgs\+\_\+optimize()}} arguments. Similarly, a user can implement a function compatible with \mbox{\hyperlink{namespacelbfgs_a0c10e9cbe6e2a62c19226cf94a5fde18}{lbfgs\+\_\+stepbound\+\_\+t}} to provide an external upper bound for stepsize, and \mbox{\hyperlink{namespacelbfgs_a65146245bda9f01d3635e84d0e92197d}{lbfgs\+\_\+progress\+\_\+t}} (progress callback) to obtain the current progress (e.\+g., variables, function, and gradient, etc) and to cancel the iteration process if necessary. Implementation of the stepbound and the progress callback is optional\+: a user can pass nullptr if progress notification is not necessary.
\end{DoxyEnumerate}

Line search method for smooth or nonsmooth functions. This function performs line search to find a point that satisfy both the Armijo condition and the weak Wolfe condition. It is as robust as the backtracking line search but further applies to continuous and piecewise smooth functions where the strong Wolfe condition usually does not hold.
